# Модуль `timeit` 

### [Назад в Профилирование ⤶](/data/profiling/profiling.md)

---

Модуль `timeit` предназначен для точного измерения времени выполнения небольших фрагментов кода (это важно при сравнении 
эффективности различных реализаций структур данных и алгоритмов).  

Модуль `timeit` предоставляет достоверные результаты (код выполняется многократно и рассчитывается среднее время 
выполнения), минимизируя влияние случайных факторов (кэширование, процессы ОС, оптимизация интерпретатора).  

## Основные методы и параметры
**timeit.timeit(stmt, setup, number, globals):**
 - `stmt` — строка с исполняемым выражением (по умолчанию `pass`).
 - `setup` — строка с подготовительным кодом, который выполняется один раз перед измерением (по умолчанию `pass`).
 - `number` — количество повторений (1000000 по умолчанию).
 - `globals` — словарь с глобальными переменными окружения, используемого при исполнении кода.
**timeit.repeat(stmt, setup, repeat, number, globals):**
 - Повторяет измерения `repeat` раз (по умолчанию 5), возвращая список результатов.

## Принципы работы
- **Изоляция кода**: код исполняется в отдельной области видимости, предотвращая влияние переменных глобального пространства имён.  
- **Многократное выполнение**: код повторяется большое количество раз (по умолчанию 1 000 000 раз для простых операций), 
что позволяет сгладить влияние случайных ошибок измерения.  
- **Использование точных функций времени**: используется функция с максимально возможной точностью (`time.perf_counter()`).  

## Примеры использования
Пример измерения времени выполнения выражения

```python
import timeit

result = timeit.timeit("sum([i for i in range(100)])", number=100000)
print(result)
```
